Mi legyen a 1337 moat-ja?

A platform? A skills ökoszisztéma? Az FDE delivery modell? Vagy a vertikális iparági tudás? — A válasz meghatározza az MVP-t.

A
"FDE-First"
Services + Platform Tooling
Palantir Playbook
"A platform az FDE munkájának a mellékterméke. Előbb oldjuk meg az ügyfél problémáját, aztán productizáljuk a megoldást. Nem platform-ot adunk el, hanem eredményt."

Miben áll az MVP

  • FDE service: 2-4 hetes engagement-ek
  • Skill-ek: amit az FDE épít az ügyfélnek
  • Platform: belső tooling, nem termék
  • Minden engagement → új skill, új MCP server

  • Mély ügyfélkapcsolatok

  • Consulting-nak tűnik, nem SaaS-nak
  • Valuation alacsonyabb lesz
  • Függőség a csapat méretétől

Javaslat: Opció C — "Vertical Deep"

A kutatás alapján az Opció C a legerősebb pozíció a 1337 számára, mert:

1. A skills-ben hiszel — és a skills akkor a legerősebbek, ha domain-specifikusak. Egy "Code Review" skill generikus, egy "SZKP Kérelem Feldolgozás" skill egyedi és lemásolhatatlan.

2. A nagy szereplők nem jönnek ide — Credal és Glean nem fognak magyar hitelezési regulációkkal foglalkozni. Az AI Act és MNB compliance lokális tudást igényel.

3. Reference > Feature — A Credal story tanulsága: "We'd write a case study, and then heads of platform engineering would reach out." Egy sikeres KAVOSZ deployment több ügyfelet hoz, mint bármilyen feature list.

4. FDE + Product hybrid — Nem kell választani. Az FDE építi a vertikális megoldást, a platform generalizálja. Ahogy Palantir tette: "Each FDE builds prototypes, the platform org generalizes."

Beépített Best Practices (kutatásból)

  • Governance first: "75% of leaders prioritize security, compliance, auditability as most critical" — építsd be az MVP-be
  • Data readiness: A Gartner szerint ez a #1 failure ok — az FDE pont ezt oldja meg
  • Multi-agent: Dust tapasztalata: "specialized agents outperform single general-purpose" — a skills modell pont ez
  • Case study driven GTM: Credal: "We'd write a case study, heads of engineering would reach out" — KAVOSZ = első case study
  • Hybrid PLG + Sales: "Land with PLG, expand with sales" — self-service demo + FDE engagement
  • Build on platform, not from scratch: MCP, Claude Code SDK, Craft Agent — ne újra feltalálni a kereket
